# Next.js App Router skill

When editing `app/` routes:

1. Decide server vs client boundary before adding hooks or browser APIs.
2. Align data fetching and caching with how sibling routes behave.
3. Reuse existing layout and error boundary patterns.

Use for new routes, RSC refactors, or metadata/SEO tweaks.
