# Repository hygiene skill

When editing this repo:

1. Touch only files needed for the request; avoid unrelated cleanup.
2. Follow existing naming, folder structure, and patterns nearby.
3. Prefer commits/PR text that explain *why* and how to verify.
4. Never add secrets or credentials to the codebase.

Invoke when the user asks for refactors spanning many files, large formatting-only changes, or vague "clean up" tasks—narrow scope first.
