# TypeScript skill

When working in `.ts` / `.tsx`:

- Fix types instead of suppressing with `any` or `@ts-ignore` unless unavoidable.
- Align with existing module layout and path aliases.
- Prefer explicit function return types on exported APIs.

Use for refactors, new modules, or type-error cleanup.
